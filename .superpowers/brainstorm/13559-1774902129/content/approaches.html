How should we build Chip In?

Three architectural approaches — each with real trade-offs

A

Native SwiftUI + Supabase

Best iOS feel, best performance. SwiftUI for the UI (buttery animations, native controls, widgets, lock screen), Supabase for real-time sync, auth, and database. Receipt scanning via iOS Vision framework. Offline via SwiftData.

✅ Smoothest iOS experience  ✅ Widgets + Shortcuts  ✅ Free tier on Supabase
❌ iOS only  ❌ Requires Swift knowledge
B

React Native + Expo + Supabase

Fast to build, cross-platform ready. Expo handles the iOS build pipeline, React Native for UI, Supabase for backend. Can add Android later with minimal extra work. Receipt scanning via Expo Camera + OpenAI Vision API.

✅ Faster dev  ✅ Android-ready later  ✅ Large ecosystem
❌ Slightly less native feel  ❌ Widgets need native module
C

Flutter + Firebase

Beautiful custom UI, cross-platform. Flutter gives pixel-perfect custom designs that look stunning. Firebase for real-time sync, auth, and push notifications. Easiest receipt scanning with ML Kit.

✅ Custom UI freedom  ✅ Real-time out of the box  ✅ iOS + Android
❌ Firebase costs scale fast  ❌ Dart language  ❌ Heavier app size
My Recommendation

Option A — Native SwiftUI + Supabase. Since you want superior iOS UI, widgets, lock screen support, and buttery smooth performance, native is the right call. Supabase is completely free at your scale and handles real-time sync, auth, and offline perfectly.
